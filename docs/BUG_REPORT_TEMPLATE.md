# Баг-репорт после тестирования opora-migration

## Окружение
- Браузер: Chromium headless (Playwright)
- Тестировал на мобильном: да (375px эмуляция)
- Проект запущен: локально (`npm run dev`)
- Дата: 18 марта 2026

---

## Критичные 🔴 (не работает функционал)

### Баг 1 — ✅ ИСПРАВЛЕН
- **Где:** /admin/contacts, /admin/news, /admin/texts
- **Что делал:** Залогинился в админку → перешёл на страницу Контакты
- **Ожидал:** Данные загрузятся и отобразятся в полях
- **Получил:** `Runtime TypeError: Cannot read properties of undefined (reading 'map')` — белый экран с ошибкой
- **Причина:** JWT-cookie был установлен с `path: "/admin"`, но API-маршруты находятся на `/api/admin/*`. Браузер не отправлял cookie на `/api/admin/contacts`, middleware возвращал 401, клиентский код не обрабатывал ошибку и пытался вызвать `.map()` на `undefined`.
- **Исправление:** Cookie path изменён на `"/"` в `src/lib/auth.ts`. Добавлена обработка ошибок fetch во всех admin-страницах (contacts, news, texts, images).
- **Скриншот:** /tmp/opora-tests/11_admin_contacts.png (до исправления)

### Баг 2 — ✅ ИСПРАВЛЕН
- **Где:** /admin/news
- **Что делал:** Залогинился → перешёл на Новости
- **Ожидал:** Список новостей
- **Получил:** `Runtime TypeError: news.map is not a function`
- **Причина:** Та же — cookie не отправлялся на API. `fetch` возвращал HTML-страницу редиректа вместо JSON, `res.json()` парсил HTML как объект, и `.map()` падал.
- **Исправление:** Добавлена проверка `Array.isArray(data)` и try/catch в `loadNews()`.
- **Скриншот:** /tmp/opora-tests/12_admin_news.png (до исправления)

## Важные 🟠 (работает, но неправильно)
