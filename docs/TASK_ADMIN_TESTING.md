# Задача: ручное тестирование админ-панели

## Цель
Проверить что админ-панель полностью работоспособна — все разделы открываются, данные сохраняются, изменения видны на публичном сайте.

## Доступ
- URL: http://localhost:3000/admin (или адрес сервера)
- Логин: `admin`
- Пароль: (спросить у разработчика или см. .env.local → ADMIN_PASSWORD)

## Как запустить локально
```bash
cd opora-migration
npm install
npm run dev
# Открыть http://localhost:3000
```

### Сценарий 3: Управление новостями (CRUD)

| Шаг | Действие | Ожидаемый результат |

| 1 | Открыть /admin/news | Список новостей (8 штук по умолчанию) |

## Формат отчёта

После прохождения заполни каждый сценарий:
- ✅ — всё работает как описано
- ❌ — есть проблема (опиши что именно не так)

Если баг — укажи:
1. На каком шаге сломалось
2. Что увидел вместо ожидаемого
3. Скриншот (если можешь)

### Пример заполнения

```
Сценарий 3: ❌
Шаг 4 — после нажатия «Сохранить» ничего не произошло,
сообщение «Сохранено!» не появилось. В консоли ошибка 500.
Скриншот: [фото экрана]
```
